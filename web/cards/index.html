  • JavaScript

    This one uses JavaScript to make the whole thing clickable, but really only the title is a link.

    The description does not show a link preview, and you cannot open in a new tab from the description (ctrl+click, middle click or right click).

  • CSS Only

    This one has the whole thing as a link.

    Screen readers will read this entire text before announcing the link. The text can only be selected by holding alt. There is no way to drag the image. It's impossible to nest links.

  • CSS + ARIA

    This one is very similar to "CSS Only," except it defines some ARIA attributes which help identify the label and the description.

    Whether or not it works is complicated.

  • Pseudo-element Trick

    This one expands its link over the full parent.

    In Chrome, it at first appears possible to select text by holding alt, but only the cursor changes. Nothing can be selected.

    It's possible to have nested links using position: relative.

  • Compromise

    This one links around the image and the title only, which preserves the majority of the clickable area.

    On Chrome the title is not draggable.

  • Compromise 2

    This one has its link inside the title only, which is more semantically correct. It has a pseudo-element covering the (container of) the image and the title, but not this paragraph (which is selectable as normal).

    Users could easily click here thinking it would work, though.

    • This last one is a little special, because it only includes an image and a title. In this case, the tree can go h2 > a > img without much trouble since img is naturally an inline element and won't show up on screen readers.

    The margins would have to be considered for this one. Flexbox could also be used to switch the order of the text and the image, and it's all part of an h2 so it should all look correct.
